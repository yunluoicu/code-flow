# CodeFlow OpenSpec

OpenSpec 规范管理。复杂需求必须通过 /opsx:propose 创建 change-id，经过 spec-review 确认规范完整性，archive 前必须 Review 无 must-fix 项。

**使用场景**：复杂需求流程的 propose → spec-review → archive 阶段。

始终使用简体中文回复。

本 Skill 只提供流程、检查项和输出规范，不自动执行 Git 写操作。
