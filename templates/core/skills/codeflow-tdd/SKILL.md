# CodeFlow TDD

测试驱动开发。有代码逻辑改动时必须先写测试再写实现。

**使用场景**：所有代码逻辑改动（简单需求、复杂需求均必须）。

**Go 要求**：table-driven tests、覆盖正常路径+边界+错误路径。
**Vue 要求**：组件测试覆盖关键交互、状态变化、边界场景。

始终使用简体中文回复。

本 Skill 只提供流程、检查项和输出规范，不自动执行 Git 写操作。
